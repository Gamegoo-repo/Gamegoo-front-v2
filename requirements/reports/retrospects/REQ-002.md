## Retrospective Report — 2026-07-04 #1

### ✅ What Went Well

- **핵심 목표 달성**: auth 도메인 상태(`useAuthStore`·`useAuth`·`loginRequiredModalStore`)와 타입을 `entities/auth/model`로 이전하고 루트 barrel(`@/entities/auth`)을 신설. 외부 30개 소비자가 전부 barrel 경유(deep import 0건).
- **연쇄 위반까지 정통 해소**: store 이전이 유발한 상향 의존을 모두 제거 — shared→entities(auth store), entities→features(post-list→board), shared→features(use-authenticated-action).
- **순수 relocation 유지**: 이전 파일 3종(`authStore`/`useAuth`/`loginRequiredModalStore`)은 원본과 `diff` 결과 로직 완전 동일. `initializeAuth`의 토큰 refresh, `logout`의 기존 불일치(`user: undefined`)까지 그대로 보존 — feature 변경 유입 없음.
- **자체 딥 리뷰로 잠복 버그 선제 발견·수정**: socket provider를 app으로 옮기며 발생한 `features→app` 상향 5건을, Context+`useGamegooSocket` 훅을 `shared/api/socket/gamegoo-socket-context.ts`로 분리해 해소. build/tsc가 못 잡는 레이어 위반을 FSD grep으로 포착.
- **검증 정합**: `pnpm build` 통과, tsc 신규 에러 0 (baseline 20 == 20), 독립 리뷰 에이전트 "behavior-changing bug 없음" 확인.

### ⚠️ What Needs Improvement

- **DI 배선 반복** — `bump-button`, `board-toolbar-desktop/mobile`, `open-chat-room-button`, `post-detail-modal` — `useAuthenticatedAction`을 순수 DI로 전환하며 `isAuthenticated`+`onUnauthenticated` 주입 3줄이 5곳에 중복. shared 순수성을 얻은 대가. 편의 래퍼는 entities 배치 논쟁이 있어 보류.
- **`any[]` 제네릭 잔존** — `shared/hooks/use-authenticated-action.ts` — `T extends (...args: any[]) => void` (원본에서 이월). biome 경고 1건. `unknown[]`로 좁히면 `Parameters<T>` 추론에 영향 가능 → 별도 검토.
- **수동 스모크 미실시** — 자동 게이트는 green이나 OAuth 콜백/게시판 모바일/소켓 재연결의 런타임 확인은 아직 안 함.
- **소켓 context 직접 경로 import** — 소비자가 `@/shared/api/socket/gamegoo-socket-context`를 직접 import (barrel 미경유). 기존 `socket-manager` 직접 import 패턴과 일관되므로 위반은 아님.
