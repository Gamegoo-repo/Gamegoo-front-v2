## Retrospective Report — 2026-05-10 #1

### ✅ What Went Well

- **case-only rename 안전 처리**: macOS case-insensitive APFS 환경에서 `git mv -f`로 한 번에 처리되어 임시 파일 단계 없이 히스토리를 보존하며 rename 완료. Linux Docker 빌드(case-sensitive) 잠재 깨짐 해소.
- **변경 영향 범위가 plan과 정확히 일치**: 영향 import 사이트 1곳(`pages/riot/index.tsx`)만 갱신. 인접 코드를 건드리지 않아 PR 노이즈 제로.
- **변경 파일 한정 검증 모두 clean**: tsc/biome/build 통과. 코드베이스 전반의 사전 존재 위반은 손대지 않음(리팩토링 범위 원칙 준수).
- **barrel 경유 정상화**: `AuthPage`라는 잘못된 별칭이 제거되고 `AuthSection`이 슬라이스 public API로 일관됨. deep import 진입점 1개 차단.

### ⚠️ What Needs Improvement

- **`Checkbox.onCheckedChange`의 `checked as boolean` 좁힘**: `src/widgets/auth/ui/AuthSection.tsx:34` — Radix `CheckedState`는 `boolean | "indeterminate"`인데 강제 캐스팅. 본 REQ 범위 밖(동작 변경 금지)이라 그대로 유지했으나, indeterminate를 사용하지 않는 의도라면 `checked === true`로 명시하는 것이 안전.
- **같은 슬라이스 내 잔존 비표준 컴포넌트**: `auth-callback.tsx`, `sign-up-section.tsx` — 둘 다 default export + kebab-case 파일명. 외부에서 deep import 중. 본 REQ가 다룬 동일 종류의 위반이며 별도 REQ로 정리해야 일관성이 완성됨.
- **`pages/riot/index.tsx`의 `useAuth` 위치**: `@/shared/model/use-auth`에서 import — 정통 위치는 `@/entities/auth/model/useAuth`. 본 REQ 범위 밖이지만 인접 비표준이며 후속 REQ 후보.
