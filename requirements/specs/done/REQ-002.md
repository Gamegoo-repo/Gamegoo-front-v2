## Summary

auth 도메인 상태(`useAuthStore`, `useAuth`)를 `shared/model/`에서 `entities/auth/model/`로 이전하고, 슬라이스 루트 `index.ts` barrel을 신설해 외부가 barrel 경유로만 접근하도록 정리한다. 동작 변경 없음.

## Background

`entities/auth`의 타입·guard·lib는 이미 올바른 위치이나 (1) 슬라이스 루트 barrel이 없어 전부 deep import이고, (2) auth store가 `shared/model/use-auth-store.tsx`에 있어 `@/entities/auth/model/types`를 import하는 shared→entities 상향 위반이 있다. auth store/hook은 auth 엔티티가 소유해야 한다.

## Requirements

- `shared/model/use-auth-store.tsx` → `entities/auth/model/authStore.ts` 이전(로직 불변, import 경로만 수정)
- `shared/model/use-auth.tsx` → `entities/auth/model/useAuth.ts` 이전
- `UserStore` 인터페이스를 `entities/auth/model/types.ts`로 이동
- `features/auth/model/login-required-modal-store.ts` → `entities/auth/model/loginRequiredModalStore.ts` 이전 (auth 도메인 UI 상태, 기존 cross-slice 소비 해소)
- `entities/auth/index.ts` barrel 신설(types, guards, parse-callback, authStore, useAuth, loginRequiredModalStore 노출)
- `useAuthStore`/`useAuth`/`UserStore`/`useLoginRequiredModalStore`의 모든 import 사이트를 `@/entities/auth` barrel 경유로 갱신
- `widgets/auth/ui/auth-callback.tsx`의 auth deep import(parse-callback, guards)도 barrel 경유로 통합
- `entities/post/ui/post-list.tsx`를 `widgets/board-view/ui/PostList.tsx`로 이전(PascalCase + named export) — useAuth cross-slice 및 기존 features 상향 위반 동시 해소
- `shared/hooks/use-authenticated-action.ts`를 순수 DI 훅으로 전환(`(action, { isAuthenticated, onUnauthenticated })`) — shared 레이어 순수성 확보. 5개 호출지가 `@/entities/auth`에서 auth 상태·모달 opener를 주입
- `shared/providers/gamegoo-socket-provider.tsx`를 **분리 이전**: Provider 컴포넌트는 `app/providers/GamegooSocketProvider.tsx`(useAuth 사용, app→entities 합법), Context+`useGamegooSocket` 훅은 `shared/api/socket/gamegoo-socket-context.ts`로 분리. features 소비자는 훅을 shared에서 import(features→shared 합법). `shared/providers` barrel에서 제거
  - 근거: Provider만 app으로 옮기면 훅 소비자(features/chat·friend)가 features→app 상향 위반이 됨. Context/훅을 shared에 두어 해소.
- `shared/model/`의 auth 2파일 삭제
